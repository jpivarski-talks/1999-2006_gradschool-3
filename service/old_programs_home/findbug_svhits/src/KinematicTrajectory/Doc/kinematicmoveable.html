The KTKinematicMoveable Class

Introduction

KTKinematicMoveable is one of the basic track classes in the CLEO3 infrastructure. It inherits directly from KTKinematicData and acts as the base class for all classes that move KTKinematicData objects. Since there is no specification of a magnetic field or material geometry in this class, all public move routines defined here are pure virtual functions, which means that KTKinematicMoveable is an abstract class and no objects of its type can be instantiated.

Note: Classes that inherit from KTKinematicMoveable must provide the required move functions.

KTKinematicMoveable class diagram

Specification of move direction and status

When you move any KTKinematicMoveable object (or objects from its descendents) you can specify the direction of the move as being forward, backard or either (the specification of "either" means that the object will take the shortest path to the desired point, either forward or backward). Forward and backward mean exactly what they say; the object will move only in the specified direction even if it has to make almost a complete turn.

The definitions of these directions can be found in the KTMoveControl class. For reasons of coding clarity it is better to typedef them as shown below:

typedef KTMoveControl::kDirectionForward  kForward;
typedef KTMoveControl::kDirectionBackward kBackward;
typedef KTMoveControl::kDirectionEither   kEither;

You can also typedef the return argument in a similar way to make your code simpler to read.

typedef KTMoveControl::MoveStatus MoveStatus;

Examples of moves can be found in the KTKinematicFixedBz class.

Which move routines are declared

Pure virtual functions are declared for the following moves

    • Fixed arc length
    • Point of closest approach to a point in 3-D
    • Point of closest approach to a point (in the bend plane)
    • Point of closest approach to a line
    • Specified z position
    • Intersection with a plane
    • Intersection with a circle of a specified radius
    • Intersection with a cylinder with specified center, axis and radius

 

KTKinematicMoveable implements all the above move routines for the case when the helix is a straight line. These functions are called automatically by the descendents of this class when they detect that the magnetic field is zero or the particle has no charge.

Other declared routines

KTKinematicMoveable also declares pure virtual functions for the point of closest approach and the distance of closest approach for the following situations. These member functions must be implemented by descendents of KTKinematicMoveable. The straight line versions of these functions have been implemented.

    • Return point of closest approach to a point in 3-D
    • Return point of closest approach to a point (in bend plane)
    • Return point of closest approach to a line
    • Return distance of closest approach to a point in 3-D
    • Return distance of closest approach to a point (in bend plane)
    • Return distance of closest approach to a line
